# ICSE Focus Plan (for submission usage only)

This round is about improving reviewer trust and first-time comprehension, not expanding research scope.

## Immediate Priorities

1. Clarify the top onboarding path (README + docs landing).
2. Keep CI trustworthy and diagnosable.
3. Ensure docs and schema are fully consistent.
4. Keep output/evidence contracts explicit and easy to validate.

## What Strengthens ICSE Now

1. A 2-3 minute project explanation for newcomers.
2. One canonical command path for reproducing the main track.
3. Clear claim taxonomy (ranking/decision/distribution only).
4. Conservative decision semantics explained once and reused everywhere.
5. Machine-checkable reproducibility chain (validate-spec, validate-evidence, compatibility checks).

## Highest Reviewer-Facing Clarity Risks

1. Too many entrypoints shown at once.
2. Ambiguous stable vs experimental surface.
3. Inconsistent claim wording across docs.
4. CI red/unclear failure causes.

## Explicitly Out of Scope Before Submission

1. New claim types or semantics.
2. Platform-level expansion (dashboards, broad APIs, plugin systems).
3. Major UI redesign work.
4. Performance-engineering projects not directly tied to paper claims.

## Deliverables for This Phase

1. README compression and canonical quickstart.
2. CI reliability hardening with better failure logs.
3. Schema/docs taxonomy alignment.
4. Release hygiene checklist (GitHub release readiness, PyPI deferred).
