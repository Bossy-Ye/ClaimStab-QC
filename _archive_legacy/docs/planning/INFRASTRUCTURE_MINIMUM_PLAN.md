# Infrastructure Minimum Plan

Goal: stay infrastructure-ready without over-productizing pre-submission.

## Minimum Surface To Keep Now

1. Installability
- `pip install -e .` with optional extras documented.

2. Stable interfaces
- CLI command/flag compatibility.
- Output artifact naming compatibility.
- Evidence schema compatibility.

3. Contribution path
- External task starter.
- Atlas publish/validate flow.
- Clear PR and dataset submission templates.

4. Reproducibility path
- Spec validation.
- Evidence validation.
- Compatibility checks.
- Deterministic baseline examples.

## Future-Adoption Signals To Preserve

1. Live Claim Explorer (advanced capability).
2. Dataset Registry / ClaimAtlas dataset flow.
3. Custom task onboarding path.
4. Evidence protocol and traceability assets.

## Post-Submission (Explicitly Postponed)

1. Platform productization and dashboard ecosystem.
2. New API guarantees beyond current contracts.
3. Major plugin architecture redesign.
4. Large UX rewrites across the docs site.

## Operating Principle

Preserve advanced/community capabilities, but keep first-time onboarding centered on paper-reproducible core workflows.
