# Governance

## Project Role Model
- **Maintainers**: review/merge PRs, release changes, and set roadmap priorities.
- **Contributors**: submit issues, PRs, and improvements.

## Decision Process
- Small changes: maintainer review + merge.
- Larger design changes: documented rationale in PR/issue and maintainer consensus.
- Breaking changes: require explicit migration notes in README/CHANGELOG.

## Project Priorities
1. Correctness and reproducibility.
2. Clear claim semantics and statistically defensible outputs.
3. Usable community tooling (docs, examples, templates, CI).

## Release Discipline
- Tag versions from `main` after tests pass.
- Record notable changes in `CHANGELOG.md`.
