# Reproduce

This page lists copy-paste commands for the current workflows in this repo.

For the canonical command-to-directory mapping, see [Output Directory Map](output_map.md).
For the locked experiment set, see [Experiment Matrix](experiment_matrix.md).
Compatibility guarantees and validation gates are in [Compatibility Contract](compatibility_contract.md).

## Output Convention (Canonical)

Preferred artifact roots for paper/submission:
- `output/paper/artifact/` (generated by `make reproduce-paper`)
- `output/presentations/core/` (core curated package)
- `output/presentations/large/` (extended curated package)

`output/exp_*` paths are supported as legacy ad-hoc locations.

## 1) Main paper track

```bash
python -m claimstab.cli validate-spec --spec paper/experiments/specs/paper_main.yml
python -m claimstab.cli run --spec paper/experiments/specs/paper_main.yml --out-dir output/presentations/large/maxcut_ranking --report
python -m claimstab.cli validate-evidence --json output/presentations/large/maxcut_ranking/claim_stability.json
```

Expected artifacts:
- `output/presentations/large/maxcut_ranking/claim_stability.json`
- `output/presentations/large/maxcut_ranking/scores.csv`
- `output/presentations/large/maxcut_ranking/robustness_map.json`
- `output/presentations/large/maxcut_ranking/rq_summary.json`
- `output/presentations/large/maxcut_ranking/stability_report.html`

## 3) Non-MaxCut BV + Atlas publication

```bash
python -m claimstab.cli validate-spec --spec examples/community/specs/atlas_bv_demo.yml
python -m claimstab.cli run --spec examples/community/specs/atlas_bv_demo.yml --out-dir output/examples/atlas_bv_demo --report
python -m claimstab.cli publish-result --run-dir output/examples/atlas_bv_demo --atlas-root atlas --contributor your_name
python -m claimstab.cli validate-atlas --atlas-root atlas
```

Expected artifacts:
- `output/examples/atlas_bv_demo/claim_stability.json`
- `atlas/submissions/<submission_id>/...`
- `atlas/index.json`

## 4) External user task starter

```bash

python -m claimstab.cli validate-spec --spec examples/my_problem_demo/spec_my_problem.yml
python -m claimstab.cli run --spec examples/my_problem_demo/spec_my_problem.yml --out-dir output/my_problem --report
python -m claimstab.cli publish-result --run-dir output/my_problem --atlas-root atlas --contributor your_name
```

## 5) Refresh website dataset page

```bash
python -m claimstab.cli export-dataset-registry --atlas-root atlas --out docs/dataset_registry.md
```

## 6) One-command paper reproduction

```bash
make reproduce-paper
```

This generates experiment outputs, HTML reports, paper figures, and a run manifest under `output/paper/artifact/`.

## Runtime Expectations
- `core` smoke runs: short.
- `standard`/`large` comprehensive runs: moderate to long.
- multi-device transpile-only: moderate (depends on number of devices and sampled configs).
- noisy simulation: longer and environment dependent.
